# 03 团队成员3 · 后端链路作战手册

> 关联：[00_协调开发规则.md](00_协调开发规则.md) · [主文档第7/9节](../工业电机故障预警与预测性维护项目开发文档.md)

## 1. 角色定位与北极星指标

- 角色：后端链路负责人（数据库直连读取 → 入库治理 → 告警路由 → API）
- 北极星指标：
  - 数据校验失败率 < 0.5%
  - 外部写库链路 24h 稳定可读
  - 告警链路端到端 < 1s

## 2. 主要交付物

- 数据接入服务（从数据库读取新增标注数据）
- 预处理与入库治理模块（字段校验、清洗、特征提取、幂等去重）
- 数据库写入层（关系型 + 向量库，幂等）
- 告警分级路由（一级/二级/低置信度分流）+ 事件流推送
- 对前端的 REST/WebSocket API
- 后端联调接口文档

## 3. 必须遵守的接口契约

来自 [00_协调开发规则.md](00_协调开发规则.md)：

```
写库字段    : device_id / node_id / timestamp / fault_class / confidence /
              health_score / vib_rms / temperature / current / need_deep_confirm
字段约束    : fault_class 枚举固定，confidence/health_score 范围 0~100
低置信度    : confidence < 70 → need_deep_confirm = 1
告警分级    : 一级温/电超限 / 二级AI识别(conf>80) / 三级深度模型 / 四级趋势
数据来源    : 外部硬件后端写入数据库（本项目不负责硬件采集与MQTT链路）
```

数据库（参照主文档第7节）：

| 表 | 用途 |
|----|------|
| devices | 设备信息 |
| edge_nodes | 节点信息 |
| annotated_data / annotated_data_raw | 原始标注数据 |
| telemetry_events | 标准事件数据 |
| alerts | 预警记录 |
| alert_events | 告警事件流 |
| prediction_records | 预测查询结果 |
| kb_documents / kb_chunks | 知识库（FULLTEXT） |
| llm_analysis | 大模型分析记录 |
| maintenance_records | 维护记录 |
| fault_vectors | 故障特征向量(32维) |

## 4. 6周任务分解

| 周次 | 主要任务 | 完成判据 |
|------|----------|----------|
| 第1周 | 字段契约对齐 + 数据校验器 + 单元测试 | 外部样例数据100%通过字段校验 |
| 第2周 | 数据库schema + 幂等写入 | 重复写入不产生重复记录 |
| 第3周 | 告警路由 + REST API骨架 | 读库稳定、告警可查 |
| 第4周 | WebSocket推送 + 前端联调 | 前端3秒内可见告警 |
| 第5周 | 压测、限流、错峰策略 | 端到端<1s，掉包<0.5% |
| 第6周 | 大模型分析接入 + 收尾 | LLM调用降级链路完整 |

## 5. 每日完成定义（DoD）

- [ ] 当日代码已commit并通过单元测试
- [ ] 当日数据校验失败率/入库延迟有日志记录
- [ ] API文档同步更新（每加一个接口）
- [ ] 告警分级符合阈值表，无私改

## 6. 自测与联调清单

后端自测：

- [ ] 非法字段数据只记录日志并隔离，不阻塞主流程
- [ ] 同 `device_id + timestamp` 幂等写入
- [ ] 低置信度样本自动打标 `need_deep_confirm = 1`
- [ ] 告警落库后可被 API 查询
- [ ] 告警事件可实时推送（WebSocket events 频道）
- [ ] 主消费线程无单点阻塞

联调（与团队成员1/2/4）：

- [ ] 与团队成员1：连续30分钟读库校验通过率100%
- [ ] 与团队成员1：重复写入场景下重复写入率<0.5%
- [ ] 与团队成员2：低置信度分流可被前端看到状态
- [ ] 与团队成员4：告警/设备列表 API 返回字段对齐

## 7. 风险点与升级路径

| 风险 | 触发条件 | 应对 | 升级人 |
|------|----------|------|--------|
| 读库阻塞 | 数据库压力过高 | 分页拉取+游标+限流 | 团队成员5登记 |
| 字段异常激增 | 外部写库口径漂移 | 立即停止部署，回滚到上一版字段映射 | 团队成员1对齐 |
| 数据库性能 | 写入抖动 | 索引/批量提交/分表 | 团队成员5记录指标 |
| LLM API故障 | 调用失败 | 降级本地模型，告警不阻塞 | 全员知会 |

## 8. 工具栈与代码路径

- 开发环境：Python 3.8+
- 关键依赖：`SQLAlchemy`、`FastAPI`、`chromadb`
- 推荐目录：`backend/`
  - `backend/protocol.py`（字段映射与校验规则）
  - `backend/db/`（schema + 迁移）
  - `backend/alert_router.py`
  - `backend/api/`（REST + WebSocket）
  - `backend/llm_bridge.py`（对接 [motor_ai_core.py](../motor_ai_core.py)）
- 配置：`.env` 存放 DB、LLM 凭证；`.env` 不入仓

> **注意**：团队成员4的 Python Dash 可视化大屏（`python_dashboard/`）也会消费本后端提供的 REST API（`/api/devices`、`/api/history`、`/api/alerts`），字段契约以本文档第3节为准，双方保持对齐。
