# 05 团队成员5 · 项目推进与测试作战手册

> 关联：[00_协调开发规则.md](00_协调开发规则.md) · [团队协作执行看板模板.md](../团队协作执行看板模板.md) · [主文档11节](../工业电机故障预警与预测性维护项目开发文档.md)

## 1. 角色定位与北极星指标

- 角色：项目推进与质量保障（看板、回归、纪要、版本日志）
- 北极星指标：
  - 当日问题清单 100% 状态更新
  - 阻塞超 24 小时 100% 升级到决策人
  - 周验收门禁全部留痕

## 2. 主要交付物

- 每日站会纪要 + 任务看板更新
- 问题清单（缺陷/阻塞/变更）
- 回归测试脚本 + 每日回归报告
- 版本变更日志（协议/阈值/接口/数据库）
- 周验收门禁结果与演示纪要
- E2E 用例执行台账（E2E-01~E2E-05）

## 3. 必须遵守的接口契约

- 看板模板：[团队协作执行看板模板.md](../团队协作执行看板模板.md)
- 阈值与协议变更：必须经评审才允许实现（[00_协调开发规则.md](00_协调开发规则.md) 第4节）
- 问题字段必填：

```
ID / 现象 / 复现步骤 / 责任人 / 截止时间 / 当前状态 / 复测结论
```

- 演示原则：周五只展示已联通链路，未联通不演示

## 4. 6周任务分解

| 周次 | 主要任务 | 完成判据 |
|------|----------|----------|
| 第1周 | 看板与问题清单上线、E2E用例落地 | 全员开始按看板执行 |
| 第2周 | 设备稳定上报回归 | 4h稳定测试报告归档 |
| 第3周 | 后端接收回归 + 接口文档审计 | 双Topic回归报告 |
| 第4周 | 前端联调回归 | 告警端到端用例通过 |
| 第5周 | 24h压测组织 + 数据收集 | 端到端<1s、可靠性>99.5% |
| 第6周 | 验收报告 + 版本归档 | 全部门禁通过、产物清单完整 |

## 5. 每日完成定义（DoD）

- [ ] 站会纪要发布（含责任人、截止时间）
- [ ] 问题清单全部更新状态
- [ ] 当日回归脚本执行并出报告
- [ ] 阻塞 > 24h 的项已升级到决策人

## 6. 回归与E2E用例（与主文档11.8一致）

| 用例ID | 目标 | 主要参与 |
|-------|------|----------|
| E2E-01 | 正常上报链路 | 团队成员1、3 |
| E2E-02 | 告警触发链路 | 团队成员2、4 |
| E2E-03 | 断线重传 | 团队成员1、3 |
| E2E-04 | 低置信度分流 | 团队成员3 |
| E2E-05 | 24h稳定性 | 全员 |

每条用例每周至少跑1次；E2E-05在第5/6周组织正式运行。

## 7. 风险点与升级路径

| 风险 | 触发条件 | 应对 | 升级 |
|------|----------|------|------|
| 信息黑洞 | 站会有人不说阻塞 | 立即结对追问 | 项目负责人 |
| 协议变更被绕过 | 没有变更单 | 拒绝合并/部署 | 项目负责人 |
| 验收延期 | 关键门禁未达标 | 周五前48h预警 | 全员 |
| 测试覆盖不足 | E2E用例减少 | 强制回归恢复 | 自负责 |
