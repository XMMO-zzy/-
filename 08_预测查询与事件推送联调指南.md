# 08 预测查询与事件推送联调指南

## 1. 启动顺序

1. 启动后端：`python -m uvicorn backend.main:app --host 127.0.0.1 --port 8000`
2. 手动触发同步：`POST /api/v1/sync/raw-to-events`
3. 启动前端：`npm run dev`

## 2. 关键接口

- 健康检查：`GET /health`
- 预测列表：`GET /api/v1/predictions`
- 设备预测：`GET /api/v1/devices/{device_id}/predictions`
- 告警事件：`GET /api/v1/events/alerts`
- 知识库写入：`POST /api/v1/kb/documents`
- 知识库查询：`POST /api/v1/kb/search`

联调前置检查（数据库）：

- 原始接入表存在：`annotated_data`（当前实现）或 `annotated_data_raw`（目标设计）
- 标准事件表存在：`telemetry_events`
- 预测表存在：`prediction_records`
- 事件表存在：`alert_events`
- 知识库表存在：`kb_documents`、`kb_chunks`（含 FULLTEXT 索引）

## 3. WebSocket 频道

- `ws://127.0.0.1:8000/ws/telemetry`：遥测推送
- `ws://127.0.0.1:8000/ws/alerts`：告警推送
- `ws://127.0.0.1:8000/ws/events`：事件推送

测试心跳：
- 发送：`{"type":"ping"}`
- 返回：`{"type":"pong"}`

## 4. 常见问题

- 预测为空：先执行 `/api/v1/sync/raw-to-events`，确认原始表有新增数据。
- 事件为空：确认规则触发条件达到告警阈值。
- 知识库无命中：检查 FULLTEXT 索引是否建立；中文库建议使用 `WITH PARSER ngram`；必要时放宽查询词。
- 大模型失败：检查 `LLM_API_URL`、`LLM_API_KEY`，并观察 fallback 输出。
