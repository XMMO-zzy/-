# 02 团队成员2 · 边缘算法作战手册

> 关联：[00_协调开发规则.md](00_协调开发规则.md) · [主文档第6节](../工业电机故障预警与预测性维护项目开发文档.md)

## 1. 角色定位与北极星指标

- 角色：边缘算法负责人（特征提取、阈值策略、分类输出）
- 北极星指标：
  - 当日样本回放通过率 ≥ 95%
  - 推理时间 < 20ms（后端本地推理基线）
  - confidence 与 health_score 输出稳定，无突变

> **部署位置说明**：ESP8266 可用内存约 80KB，无法运行 TFLite 模型。边缘算法分两层部署：
> - **采集端（ESP8266）**：时域特征计算（RMS、Peak、Kurtosis、CrestFactor）+ 256点FFT主频，结果随帧上报
> - **后端本地推理**：收到特征数据后，用轻量分类器（规则引擎或量化模型）完成 FaultClass / Confidence / HealthScore 推理，延迟 < 20ms，仍属于"近边缘推理"
>
> 对外仍称"边缘AI"，三层协同链路不变：本地推理（<20ms）→ 深度模型（<100ms）→ 大模型（秒级）

## 2. 主要交付物

- 时域特征模块（RMS、峰值、峭度、波峰因子）—— C 实现，部署在 ESP8266 固件
- 频域特征摘要（256点FFT主频）—— C 实现，部署在 ESP8266 固件
- 后端本地推理模块（规则引擎 + 轻量分类器）—— Python 实现，部署在后端
- 分类标签输出（fault_class / confidence / health_score）
- 边缘自测报告（混淆矩阵、推理耗时分布）

## 3. 必须遵守的接口契约

分类标签固定枚举（不可扩展，需扩展走变更流程）：

```
normal | bearing_fault | winding_fault | rotor_fault | overload
```

输出字段约束（与协议保持一致）：

| 字段 | 范围 | 说明 |
|------|------|------|
| FaultClass | 0~4 | 对应上面5种分类 |
| Confidence | 0~100 | 置信度，整数 |
| HealthScore | 0~100 | 健康分，整数 |

阈值（V1.0冻结）：

```
高置信度告警 : confidence > 80 且 fault_class != normal
低置信度分流 : confidence < 70 自动上送后端二次确认
温度上限     : 80 °C
电流上限     : 120% 额定
```

## 4. 6周任务分解

| 周次 | 主要任务 | 完成判据 |
|------|----------|----------|
| 第1周 | 时域特征模块（Python离线验证 + C固件实现）+ 单元测试 | 离线样本RMS/峰值/峭度计算正确 |
| 第2周 | FFT频域摘要 + 数据质量评估 | 256点FFT主频偏差<1Hz |
| 第3周 | 后端本地推理模块（规则引擎 + 轻量分类器） | 离线准确率>=85%，推理<20ms |
| 第4周 | 设备侧特征计算集成 + 后端推理联调 | 输出三字段稳定，回放通过率>=95% |
| 第5周 | 阈值与误报压测 | 噪声场景误报率<5% |
| 第6周 | 模型版本归档与说明 | model_version上线、文档冻结 |

## 5. 每日完成定义（DoD）

- [ ] 当日特征/模型修改有单元测试覆盖
- [ ] 当日离线样本回放报告（准确率、推理时间）
- [ ] 阈值或标签调整必须先评审再实现
- [ ] 输出稳定性验证（连续运行无突变）

## 6. 自测与联调清单

边缘算法自测：

- [ ] 时域特征：RMS、Peak、Kurtosis、CrestFactor 与参考值偏差<1%
- [ ] 频域：主频偏差<1Hz
- [ ] 推理耗时：均值<20ms，P99<30ms
- [ ] 输出稳定：连续30分钟无突变
- [ ] 边界条件：满量程、零信号、缺数据均不崩溃

联调（与团队成员1、3）：

- [ ] 与团队成员1：算法输出可被打包模块正确序列化
- [ ] 与团队成员3：低置信度样本能进入后端深度分析队列
- [ ] 与团队成员3：高置信度样本能触发alert Topic
- [ ] 误报样本回看可被记录、可复现

## 7. 风险点与升级路径

| 风险 | 触发条件 | 应对 | 升级人 |
|------|----------|------|--------|
| 量化精度损失 | 模型量化后准确率掉幅>5% | 重新选择特征/调阈值 | 团队成员5登记 |
| 阈值漂移 | 现场误报率突增 | 触发阈值评审 | 团队成员3联调验证 |
| 推理超时 | 个别样本>50ms | 简化模型/优化FFT | 团队成员1协助 |
| 标签缺失 | 出现非枚举标签 | 立即回滚，触发变更流程 | 团队成员5主持评审 |

## 8. 工具栈与代码路径

- 开发环境：Python 3.8+（离线训练 + 后端推理）、C/C++（设备端特征计算）
- 推荐目录：`edge_algo/`
  - `edge_algo/features.py`（离线特征验证）
  - `edge_algo/train.py`（轻量模型训练）
  - `edge_algo/quantize.py`（INT8量化，备用）
  - `edge_algo/local_inference.py`（**后端本地推理模块**，规则引擎 + 分类器）
  - `edge_algo/features.c`（设备端时域特征计算，供固件调用）
  - `edge_algo/test_samples/`（离线样本）
- 数据集：开源工业电机故障集 + 现场采集
- 模型版本：`edge_v1.0 → edge_v1.1`，写入帧的`Version`字段
