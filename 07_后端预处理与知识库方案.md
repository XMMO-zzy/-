# 07 后端预处理与知识库方案

## 1. 改造目标

- 采集端仅上传原始测点，不再承担预处理。
- 后端完成清洗、特征提取、标准化入库。
- 无向量库条件下，使用 MySQL FULLTEXT 构建首版知识库检索。
- 知识检索结果作为大模型上下文，提升解释与建议质量。

## 2. 后端预处理流程

1. 读取 `annotated_data` / `annotated_data_raw` 新增记录（当前实现优先 `annotated_data`）。
2. 解析 `raw_json` 与 `diagnosis_result`。
3. 提取关键特征：`rms / peak / mean / std`，计算数据质量分。
4. 标准化写入 `telemetry_events`。
5. 基于事件生成 `prediction_records` 与 `alerts`。

## 3. 知识库流程（FULLTEXT）

1. 文档写入 `kb_documents`。
2. 文档切片写入 `kb_chunks`（固定长度切片）。
3. 在 `kb_chunks.content` 建立 FULLTEXT 索引（中文建议 `WITH PARSER ngram`）。
4. 查询时 `MATCH ... AGAINST` 召回 TopK。
5. 召回结果拼接为 `kb_context` 传入 LLM。

## 4. 大模型调用规范

- 输入：设备状态 + 预测信息 + `kb_context`
- 输出：结论、风险概率、建议动作
- 配置：`GENERAL_LLM_PROVIDER`、`GENERAL_LLM_MODEL`、`LLM_API_KEY`

## 5. 与前端联调契约

- 实时遥测：`/ws/telemetry`
- 实时告警：`/ws/alerts`
- 实时事件：`/ws/events`
- 查询接口：
  - `GET /api/v1/predictions`
  - `GET /api/v1/events/alerts`
  - `POST /api/v1/kb/search`
