# 01 团队成员1 · 设备链路作战手册

> 关联：[00_协调开发规则.md](00_协调开发规则.md) · [主文档第4/5节](../工业电机故障预警与预测性维护项目开发文档.md)

## 1. 角色定位与北极星指标

- 角色：设备链路负责人（ESP8266 + HC-05 + 二进制上报）
- 北极星指标：
  - 设备连续运行 4 小时不死机
  - 上报帧 CRC 通过率 100%
  - 断线重连恢复时间 < 30 秒

> **采集端职责说明**：ESP8266 资源有限（可用内存约 80KB），采集端负责传感器采集、时域特征计算和数据打包上报。FaultClass / Confidence / HealthScore 三字段由**后端本地边缘推理模块**填充后写入数据库，采集端上报帧中这三字段填 0（占位），后端解包后立即完成推理并补全。

## 2. 主要交付物

- ESP8266 固件源码与 OTA 镜像
- 二进制协议打包模块（含 CRC16 校验）
- 时域特征计算模块（RMS、Peak、Kurtosis、CrestFactor，256点滑动窗口）
- WiFi 重连 + Flash 环形缓冲区
- HC-05 串口透传调试通道
- 设备侧自测报告（CRC通过率、丢包统计、稳定性）

## 3. 必须遵守的接口契约

来自 [00_协调开发规则.md](00_协调开发规则.md) 第2、3节：

```
帧头        : 0xAA55
Version     : 1
MsgType     : 0x01 telemetry / 0x02 alert
NodeId      : uint16
DeviceId    : uint16
Timestamp   : uint32 (Unix秒)
FaultClass  : uint8  ← 采集端固定填 0，由后端推理后补全
Confidence  : uint8  ← 采集端固定填 0，由后端推理后补全
HealthScore : uint8  ← 采集端固定填 0，由后端推理后补全
VibRms      : int16 (×1000)
Temp        : int16 (×100)
Current     : int16 (×100)
CRC16       : Modbus CRC16
```

- Topic：`motor/{device_id}/telemetry`（QoS 0）、`motor/{device_id}/alert`（QoS 1）
- `device_id` 格式：`MOTOR-XXX`（大写三位编号）
- 字节序与缩放倍率不允许改动；改动需走变更流程
- FaultClass / Confidence / HealthScore 由后端本地推理模块在解包后立即填充，采集端无需实现推理逻辑

## 4. 6周任务分解

| 周次 | 主要任务 | 完成判据 |
|------|----------|----------|
| 第1周 | 协议打包模块 + CRC16；可发出样例帧 | 样例帧能被团队成员3 100% 解包 |
| 第2周 | 集成传感器采集 + 时域特征计算 + 重连/缓存 | 连续4h稳定上报；断线<30s恢复 |
| 第3周 | 联调 telemetry/alert 双Topic | 双Topic稳定消费、无重复风暴 |
| 第4周 | 配合前端校准刷新与延迟 | 端到端<1s |
| 第5周 | 24h压测、丢包率统计 | 丢包率<0.5% |
| 第6周 | 固件版本归档、文档冻结 | OTA可回滚、版本日志归档 |

## 5. 每日完成定义（DoD）

- [ ] 当日代码已commit并打tag
- [ ] 当日CRC通过率、丢包率写入自测日志
- [ ] 阻塞项已发到站会群并登记问题清单
- [ ] 协议字段无私改

## 6. 自测与联调清单

设备侧自测：

- [ ] 帧字段顺序与缩放倍率与协议一致
- [ ] FaultClass / Confidence / HealthScore 固定填 0，不做推理
- [ ] 时域特征（RMS、Peak、Kurtosis、CrestFactor）计算正确，与离线参考值偏差 < 1%
- [ ] CRC16 通过率 100%
- [ ] WiFi 断开后本地缓存可重传，无重复风暴
- [ ] HC-05 仅作调试，不走主链路
- [ ] 异常重启后能自动恢复连接

联调（与团队成员3）：

- [ ] 回放样例帧 → 后端解包成功
- [ ] 真机连发30分钟 → telemetry 链路无中断
- [ ] 断线5分钟恢复 → 缓存数据回灌成功
- [ ] 一级阈值触发 → alert Topic推送成功

## 7. 风险点与升级路径

| 风险 | 触发条件 | 应对 | 升级人 |
|------|----------|------|--------|
| WiFi 干扰 | 频繁掉线 | 信道规划、AP重选 | 团队成员5登记 |
| Flash 写穿 | 缓存写入过频 | 调整缓存粒度、批量重传 | 团队成员3一起评审 |
| 时间不准 | NTP失败 | 增加NTP重试、本地RTC兜底 | 团队成员3 |
| 协议漂移 | 字段被改 | 立即停手，触发变更流程 | 团队成员5主持评审 |

## 8. 工具栈与代码路径

- 开发环境：Arduino IDE / PlatformIO（ESP8266 Core）
- 推荐目录：`firmware/`
  - `firmware/main.ino`
  - `firmware/protocol.h`（二进制协议）
  - `firmware/crc16.cpp`
  - `firmware/features.cpp`（时域特征计算：RMS/Peak/Kurtosis/CrestFactor）
  - `firmware/wifi_mgr.cpp`
  - `firmware/buffer.cpp`（环形缓存）
  - `firmware/bt_debug.cpp`（HC-05调试）
- 调试工具：MQTT Explorer、串口助手、Wireshark
- 版本号：`firmware_ver = 主.次.修订`，每次发版升修订位
