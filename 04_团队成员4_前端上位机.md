# 04 团队成员4 · 前端与数据可视化作战手册

> 关联：[00_协调开发规则.md](00_协调开发规则.md) · [主文档第9节](../工业电机故障预警与预测性维护项目开发文档.md)

## 1. 角色定位与北极星指标

- 角色：前端与数据可视化负责人（统一 React Web 大屏、告警闭环、报表）
- 北极星指标：
  - 数据刷新 < 1s
  - 告警生成到弹窗显示 < 3s
  - 告警可确认/关闭/追溯到原始报文
  - React 大屏可独立演示完整预警与预测性维护链路

## 2. 主要交付物

**Web 前端（React）：**
- 实时看板（重构为大屏布局：中心为工厂三维园区总览，四周环绕监控指标与图表）
- 工厂三维园区图（精确到车间级，实时叠加设备健康热力图与告警锚点）
- 告警中心（按等级/来源/设备筛选；支持确认、关闭、追溯；含预测维护来源）
- 设备详情页（最新状态、近24h告警、维护建议）
- 报表导出（按日/周导出告警与健康趋势）
- 前端API约定文档（与团队成员3共维护）

**可视化增强（并入 React，来自 Dash 原型验证）：**
- 设备健康状态矩阵（大屏风格卡片，含 RUL / 劣化速率）
- 预测性维护指标区（剩余寿命、劣化速率、提前预警天数、趋势判定）
- 三层 AI 流水线可视化（边缘AI → 深度模型 → 大模型，含状态高亮）
- 带宽对比面板（标注摘要 vs 传统全量，实时节省比例）
- 故障类型分布 + 告警等级统计（面板化）

> Dash 仅作为视觉原型与布局验证工具，所有生产功能统一在 `frontend/` 交付。

## 3. 必须遵守的接口契约

- 数据来源：团队成员3提供的 REST + WebSocket
- 告警状态机：

```
new → acknowledged → resolved
                 ↘ ignored
```

- 等级与来源映射（与主文档10.1一致）：

| 级别 | 来源 | UI颜色建议 |
|------|------|------------|
| 一级（即时） | 远端节点 | 红色 |
| 二级（边缘AI） | 远端节点 | 橙色 |
| 三级（深度分析） | 后端深度分析 | 黄色 |
| 四级（趋势） | 大模型 | 蓝色 |

- 字段命名以后端字典为准，前端不私改字段名

## 4. 6周任务分解

| 周次 | 主要任务 | 完成判据 |
|------|----------|----------|
| 第1周 | 项目初始化 + UI骨架（重构为基于参考图的工业大屏布局） | 居中三维园区总览/看板/告警/设备页骨架完成 |
| 第2周 | 实时数据接入 + 中心工厂三维地图交互与热力图 | 数据刷新<1s，地图可点击交互 |

| 第4周 | 设备详情 + 报表导出 | 报表字段与库一致 |
| 第5周 | 性能优化 + 长稳测试 | 4h无内存泄漏 |
| 第6周 | UX 细节优化 + 验收 | 主文档11.3指标达标 |

## 7. 风险点与升级路径

| 风险 | 触发条件 | 应对 | 升级人 |
|------|----------|------|--------|
| 字段漂移 | 后端API字段改动 | 立即停止合并，对齐后再发版 | 团队成员3对齐 |
| 数据风暴 | 告警在某秒内激增 | 前端节流+告警合并 | 团队成员5记录 |
| 状态机错乱 | 越级跳转 | 加守卫和单测 | 团队成员5回归 |
| 浏览器兼容 | 老版本渲染异常 | 限定支持版本 | 全员知会 |

## 8. 工具栈与代码路径

**Web 前端（主战场）：**
- 推荐栈：React 18 + Vite + TypeScript + ECharts 5 + Ant Design 5
- 推荐目录：`frontend/`
  - `frontend/src/pages/dashboard/`
  - `frontend/src/pages/alerts/`
  - `frontend/src/pages/devices/`
  - `frontend/src/pages/reports/`
  - `frontend/src/api/`（与后端对齐的API层）
  - `frontend/src/store/`（实时数据流）

**Python Dash（原型层，仅供参考）：**
- 用途：快速验证大屏视觉布局、演示脚本与指标呈现方式（不纳入生产维护）
- 目录：`python_dashboard/`（原型，不进主分支生产发版）
- 迁移规则：原型中验证通过的图表与交互，须复刻到 `frontend/` React 组件，使用 ECharts + Ant Design 实现

- 配置：API地址均走 `.env`，不允许硬编码
